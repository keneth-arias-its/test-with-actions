# Test con Actions

_Proteggi il tuo codice con unit test automatizzati e report di copertura._

## Benvenuto

- **A chi è rivolto**: Studenti, sviluppatori, Ingegneri DevOps, nuovi utenti GitHub, Ingegneri del Testing
- **Cosa imparerai**: Cos'è la continuous integration, come creare un workflow con GitHub Actions per i test, e come imporlo.
- **Cosa costruirai**: Imporremo la copertura dei test su codice Python di esempio per eseguire i test contro diverse versioni di Python in parallelo e produrre un report in una pull request, proteggendo la nostra versione di produzione.
- **Prerequisiti**:
  - [Hello GitHub Actions](https://github.com/skills/hello-github-actions)
- **Durata**: Questo esercizio richiede meno di 60 minuti per essere completato.

In questo corso:

1. Aprirai ed eseguirai test in un progetto di esempio esistente
2. Aggiungerai un workflow di test tramite l'interfaccia web di GitHub
3. Aggiungerai un workflow di copertura nell'editor di codice
4. Indagherai e correggerai un test fallito
5. Imporrai il passaggio dei test prima di unire le modifiche al codice

### Come iniziare questo esercizio

1. Fai clic col tasto destro su **[Step 0](../../actions/workflows/0-start-exercise.yml)** e aprilo in una nuova scheda.
2. Fai clic sul pulsante **Run workflow**.
3. Seleziona il branch `main` e fai clic sul pulsante verde **Run workflow**.
4. Attendi circa 20 secondi e aggiorna questa pagina.
